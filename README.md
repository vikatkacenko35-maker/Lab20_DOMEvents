## Краткое описание работы

В ходе лабораторной работы были изучены следующие темы работы с **DOM (Document Object Model)** в JavaScript:

- **Что такое DOM** — объектная модель HTML-документа, представленная в виде древовидной структуры
- **Поиск элементов DOM** — `getElementById()`, `querySelector()`
- **Изменение содержимого и стилей** — `textContent`, `style`
- **Обработка событий** — `addEventListener()`, событие `click`
- **Работа с элементами формы** — получение значений из `input`, обработка ввода
- **Валидация данных формы** — проверка обязательных полей, диапазонов значений
- **Динамическое создание элементов** — `createElement()`, `appendChild()`, делегирование событий

Также проведено сравнение подходов к работе с UI в **C# (WinForms/WPF)** и **JavaScript (DOM)**.

## Сравнение с C# (WinForms/WPF)

### DOM в JS = Controls в C# WinForms

| Концепция | C# (WinForms) | JavaScript (DOM) |
|-----------|---------------|------------------|
| **Найти элемент** | `Button myButton = this.Controls["myButton"]` | `const myButton = document.getElementById("myButton")` |
| **Изменить текст** | `label1.Text = "Новый текст"` | `label1.textContent = "Новый текст"` |
| **Добавить обработчик** | `button1.Click += HandleClick` | `button.addEventListener("click", handleClick)` |
| **Создать элемент** | `Button btn = new Button()` | `const btn = document.createElement("button")` |
| **Добавить в контейнер** | `panel1.Controls.Add(btn)` | `panel.appendChild(btn)` |
| **Скрыть элемент** | `button1.Visible = false` | `button.style.display = "none"` |

## Главные отличия JavaScript DOM от C# WinForms

| Аспект | C# WinForms | JavaScript DOM |
|--------|-------------|----------------|
| **Компиляция** | Да (в IL-код) | Нет (интерпретация) |
| **Типизация** | Строгая | Динамическая |
| **Ошибки** | Во время компиляции | Во время выполнения |
| **UI-дизайнер** | Есть (визуальный, drag & drop) | Нет (только код) |
| **Платформа** | Только Windows | Любой браузер (кроссплатформенно) |
| **Обновление приложения** | Требуется переустановка или перекомпиляция | Обновил файлы — всё работает |
| **Стилизация** | Ограниченная (свойства控件) | CSS (мощные возможности) |
| **Анимации** | Сложно реализовать | Просто (CSS-анимации) |
| **Доступ к файловой системе** | Полный доступ | Ограниченный (песочница браузера) |
| **Интеграция с ОС** | Глубокая (Windows API) | Отсутствует |

## Преимущества DOM перед WinForms

- **Кроссплатформенность** — работает везде (Windows, macOS, Linux, мобильные устройства)
- **Не нужна установка приложения** — достаточно браузера
- **Легко обновлять** — обновил файлы на сервере, и пользователи видят новую версию
- **Современные UI** — CSS, анимации, адаптивный дизайн
- **Доступность** — не требует прав администратора
- **Бесплатное распространение** — не нужно платить за лицензии

## Преимущества WinForms перед DOM

- **Визуальный дизайнер** — drag & drop элементов формы
- **Ошибки на этапе компиляции** — меньше runtime-ошибок
- **Интеграция с Windows API** — доступ к файловой системе, реестру, устройствам
- **Более понятная структура** для разработчиков с опытом desktop-разработки
- **Производительность** — нативные компоненты Windows
- **Безопасность типов** — компилятор не пропустит многие ошибки

**Вывод:** DOM и WinForms решают схожие задачи (создание пользовательского интерфейса и реакция на действия пользователя), но разными подходами. DOM более гибкий, современный и кроссплатформенный, подходит для веб-приложений. WinForms более безопасный (за счёт компиляции) и глубоко интегрированный с Windows, подходит для desktop-приложений под управлением Windows.
